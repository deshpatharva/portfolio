Core Components

Every neural network is built from six fundamental building blocks. Understanding each one makes the whole system less of a black box.

01
□

Layers

A layer is an organized group of neurons that transforms its inputs into outputs. Networks stack multiple layers so that each level can learn increasingly abstract representations of the data.

Input Layer

Receives raw data — pixel values, numbers, embeddings. No computation happens here; it simply passes features into the network.

Hidden Layers

One or more intermediate layers where feature learning occurs. Deeper networks (more hidden layers) can represent more complex patterns.

Output Layer

Produces the final prediction — a class label, a probability, or a continuous value — depending on the task.

Playground observation Adding a third hidden layer on the spiral dataset reduced training loss noticeably, but adding a fourth gave diminishing returns and started overfitting.
02
◯

Neurons

A neuron (or node) is the atomic unit of computation. It takes a weighted sum of all its inputs, adds a bias term, and passes the result through an activation function to produce a single output value.

z = w₁x₁ + w₂x₂ + … + b
output = σ(z)
w = weights    x = inputs    b = bias    σ = activation fn
Playground observation Increasing neurons per layer from 2 to 6 on the circle dataset allowed the boundary to become smoother and more accurate without the training time penalty of adding more layers.
03
≃

Weights

Weights are the learnable parameters — real numbers attached to each connection between neurons. They determine how much influence one neuron has on the next. Training a network is fundamentally the process of finding the right weights.

04
ƒ

Activation Functions

Without an activation function, a neural network collapses to a single linear transformation no matter how many layers it has. Activation functions introduce non-linearity, giving the network the ability to learn complex, curved decision boundaries.

ReLU

max(0, x) — outputs 0 for negatives, x otherwise. Fast and sparse; default choice for most hidden layers.

Sigmoid

1/(1+e⁻ₓ) — squashes output to (0, 1). Used in output layers for binary classification probability.

Tanh

Scaled sigmoid ranging from -1 to 1. Zero-centered, which can make gradient updates more symmetric.

Playground observation Switching from Tanh to ReLU on the spiral dataset converged faster (fewer epochs to the same loss), but ReLU was more sensitive to the learning rate and would occasionally stall with a bad initialization.
05
▼

Loss Functions

The loss function measures how wrong the network's prediction is. It produces a single number representing the gap between the predicted output and the true target. Training is the process of minimizing this number by adjusting weights.

Mean Squared Error (MSE)
L = (1/n) Σ (yₖ − ŷₖ)²

Penalizes large errors heavily. Used for regression tasks where the output is a continuous number.

Cross-Entropy Loss
L = −Σ yₖ log(ŷₖ)

Measures divergence between predicted probabilities and true class labels. Standard choice for classification.

Good prediction
Low loss
Poor prediction
High loss
Playground observation On the Playground's classification tasks, the test loss curve diverging above the training loss curve was a clear signal the model had started overfitting — time to reduce neurons or add noise.
06
▶

Optimization Algorithms

Optimization algorithms define how weights are updated after each pass through the data. They use gradients (the slope of the loss surface) to step the weights in the direction that reduces the loss.

SGD
Stochastic Gradient Descent

Updates weights using the gradient from a single sample or small batch. Simple but sensitive to learning rate. Can get stuck in local minima.

w ← w − η ∇L
Adam
Adaptive Moment Estimation

Maintains per-parameter learning rates that adapt based on the history of gradients. Converges faster and is more robust to hyperparameter choice than plain SGD.

w ← w − η · m̂ / (√v̂ + ε)
RMSProp
Root Mean Square Propagation

Scales the learning rate by a running average of recent gradient magnitudes. Well-suited for non-stationary problems like recurrent networks.

w ← w − η / √E[g²] · g
Playground observation With a high learning rate, SGD would oscillate and fail to converge on the spiral dataset. Reducing the rate helped, but Adam-style adaptive methods handle this automatically and reached lower loss in fewer steps.

Summary & Key Insights

Why visualizing neural networks matters, and what the Playground experiments revealed.

🏛

Architecture shapes capability

The number of layers and neurons per layer directly determines what patterns a network can represent. Simple datasets need simple architectures. Adding depth beyond what the problem requires leads to overfitting and slower training.

⚙

Weights carry all the knowledge

After training, the network's entire learned knowledge lives in its weights. The architecture is just a scaffold. Two networks with identical architectures but different weights represent completely different functions.

📈

Loss is the compass

Watching training vs. test loss together tells a complete story: both falling means the model is learning. Training falling while test rises means overfitting. Both staying high means underfitting (not enough capacity or training).

🎯

Activation functions unlock complexity

Linear activations make a deep network equivalent to one layer. Non-linear activations like ReLU are what give depth its power. Choosing the wrong one can cause gradients to vanish (Sigmoid in deep nets) or dead neurons (ReLU with bad init).

🔄

Optimization is not one-size-fits-all

The choice of optimizer and learning rate can be the difference between fast convergence and an unworkable model. Adaptive methods like Adam reduce the need for manual tuning but still reward careful hyperparameter choice.

💡

Visualization accelerates understanding

Seeing the decision boundary update in real time on the Playground made abstract concepts like overfitting, underfitting, and feature importance immediately intuitive. Reading about these effects is not the same as watching them happen.

Reflection

I went into the Neural Network Playground expecting it to confirm things I already understood from my ML coursework. It mostly did, but a few things caught me off guard.

The biggest one was feature engineering. I kept reaching for more neurons and layers when the real fix was just adding x₁² or x₂² as an input feature. Once I did that, even a shallow network solved problems that a deeper one was struggling with. It reminded me of something I see in my data engineering work: cleaning and shaping the data right often matters more than tuning the system consuming it.

The loss curves were also more useful than I expected. Watching training loss drop while test loss climbed was a cleaner signal than any metric I usually track in production. In my pipeline work I rely on counts, latency, and error rates. Seeing a model equivalent made me think about whether similar curve-watching could apply to streaming job performance over time.

Building this presentation forced me to write out definitions I thought I knew. Activation functions were the hardest to explain clearly without just restating the formula. The Playground made the intuition real: a network with linear activations draws straight lines no matter how many layers you add. Watching that change the moment you switch to ReLU was worth more than any textbook paragraph.

I'll carry the loss curve habit forward. It's a simple tool and an honest one.
