Algorithm Deep-Dives

Each card covers type, domains, use cases, and a plain-language explanation.

01

Linear Regression

Supervised
Tabular
How It Works

Fits a straight line (or hyperplane) through data by minimising the sum of squared differences between predicted and actual values. The result is a set of weights (coefficients) that map inputs to a continuous output.

Real-World Use Cases
  • House price prediction from size, location, age
  • Sales forecasting from ad spend
  • Medical outcome estimation (e.g., blood-pressure change)
ŷ = β₀ + β₁x₁ + … + βₙxₙ
02

Logistic Regression

Supervised
Tabular NLP
How It Works

Applies a sigmoid function to a linear combination of inputs, squashing output to [0, 1] so it can be interpreted as a class probability. A threshold (typically 0.5) converts probability to a binary label.

Real-World Use Cases
  • Email spam detection
  • Credit default prediction
  • Disease diagnosis (malignant vs. benign)
P(y=1) = 1 / (1 + e−z)
03

Decision Trees

Supervised
Tabular
How It Works

Recursively splits data on the feature that maximises information gain (or reduces Gini impurity). Produces a human-readable tree of if-then rules. Prone to overfitting without pruning.

Real-World Use Cases
  • Customer churn prediction
  • Loan eligibility decisions
  • Medical triage / symptom checking
Gini = 1 − Σ pᵢ²
04

Random Forest

Supervised
Tabular
How It Works

An ensemble of decision trees trained on random subsets of data (bagging) and random subsets of features. Predictions are aggregated by majority vote (classification) or averaging (regression), which dramatically reduces variance.

Real-World Use Cases
  • Fraud detection in financial transactions
  • Stock market trend prediction
  • Remote sensing / land classification
ŷ = mode{ T₁(x), T₂(x), …, Tₙ(x) }
05

Support Vector Machine

Supervised
Tabular Computer Vision
How It Works

Finds the maximum-margin hyperplane that separates classes. Support vectors are the closest data points that define the margin. The kernel trick (RBF, polynomial) allows non-linear separation in higher-dimensional spaces.

Real-World Use Cases
  • Handwritten digit recognition (pre-deep-learning era)
  • Bioinformatics — protein classification
  • Face detection pipelines
max w,b 2/‖w‖   s.t. yᵢ(w·xᵢ+b) ≥ 1
06

Neural Networks (MLP)

Supervised
Tabular NLP Generative AI
How It Works

Stacked layers of neurons with learnable weights. Each neuron computes a weighted sum of its inputs, applies an activation function (ReLU, sigmoid), and passes results forward. Back-propagation adjusts weights by gradient descent to minimise loss.

Real-World Use Cases
  • Tabular regression & classification (banking, retail)
  • Foundation of virtually all modern deep-learning systems
  • Recommendation engines
a(l) = σ(W(l)a(l-1) + b(l))
07

Convolutional Neural Network

Supervised
Computer Vision Generative AI
How It Works

Uses learnable filters (kernels) that slide over an image to extract spatial features — edges, textures, shapes — in early layers and complex objects in deeper layers. Pooling layers reduce spatial resolution and add translation invariance.

Real-World Use Cases
  • Medical imaging — tumour detection in X-rays/MRIs
  • Self-driving car perception
  • Image style transfer & generative art (backbone of GANs)
(f * g)(t) = Σ f(τ) g(t−τ)
08

RNN / LSTM

Supervised
NLP Generative AI
How It Works

Processes sequences by maintaining a hidden state passed from step to step. LSTMs add gating mechanisms (forget, input, output gates) that control information flow, solving the vanishing-gradient problem that plagued basic RNNs.

Real-World Use Cases
  • Machine translation (pre-Transformer era)
  • Time-series forecasting — energy demand, stock prices
  • Speech recognition & synthesis
hₜ = tanh(Wₕhₜ₋₁ + Wₓxₜ + b)
09

K-Means Clustering

Unsupervised
Tabular Computer Vision
How It Works

Partitions n data points into k clusters by iteratively assigning each point to the nearest centroid, then recomputing centroids as the mean of assigned points. Converges when assignments no longer change.

Real-World Use Cases
  • Customer segmentation for targeted marketing
  • Document / topic clustering
  • Image colour quantisation & compression
min Σ Σ ‖xᵢ − μₖ‖²
10

Generative Adversarial Network

Unsupervised
Computer Vision Generative AI NLP
How It Works

Two neural networks compete in a minimax game: a Generator creates fake data, a Discriminator tries to distinguish real from fake. The generator improves until it produces data indistinguishable from real samples.

Real-World Use Cases
  • Photorealistic image synthesis (faces, scenes)
  • Data augmentation for rare medical conditions
  • Deepfake generation & detection research
  • Drug molecule design in bioinformatics
minG maxD 𝔼[log D(x)] + 𝔼[log(1−D(G(z)))]

Reflection

Building this visual framework deepened my understanding of how machine learning algorithms differ not just in their mathematics, but in their inductive biases — the assumptions each method bakes in about the world.

What struck me most was the progression from interpretable models like Linear Regression and Decision Trees, where every decision can be traced, to black-box deep learning models like CNNs and GANs that trade interpretability for raw expressive power. Mapping each algorithm to application domains made this tradeoff tangible: for structured tabular data with limited samples, a well-tuned Random Forest often outperforms a neural network; for raw pixels or sequences, deep architectures are unmatched.

The emergence of Generative AI as a fourth domain was a key learning. GANs, LSTMs, and Neural Networks all contribute to generative pipelines, and understanding their individual mechanics makes the current wave of large generative models — diffusion models, transformers, LLMs — easier to reason about as combinations and refinements of these foundational ideas.

This artifact will serve as a living reference I return to throughout my studies, particularly when deciding which algorithm to reach for in a new problem context.
